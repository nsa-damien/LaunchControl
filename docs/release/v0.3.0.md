# Release v0.3.0

**Release Date:** 2026-02-15

## Highlights

LaunchControl v0.3.0 adds a full plist editor for viewing and editing launch item configurations directly in the app. Double-click any launch item or use the new Edit context menu to open a structured form editor with sections for Program, Schedule, Environment, I/O, and Advanced settings, plus a raw XML tab for inspecting the full plist source.

## What's New

- **Plist Editor** - Double-click or right-click > Edit to open any launch item's configuration
- **Structured Form** - Edit Program arguments, environment variables, calendar schedules, I/O paths, and advanced settings through a clean form interface
- **Raw XML View** - Inspect and edit the full plist XML source directly
- **Read-Only Mode** - System agents and daemons are shown in read-only mode for safety
- **Save with Reload** - When saving changes to a running agent, prompts to reload for changes to take effect
- **Edit Context Menu** - Quick access to the editor from the right-click context menu
- **Reveal in Finder** - Navigate to any launch item's plist file in Finder
- **Makefile** - Convenient `make build`, `make test`, `make archive`, and `make package` targets
- **CI/CD Pipeline** - GitHub Actions workflows for PR checks and automated release builds
- **Test Suite** - Unit tests for ViewModel filtering, plist validation, and agent installation

## Bug Fixes

- Calendar interval picker labels now display correctly on macOS
- launchctl enable/bootstrap failures surface to user via error message
- Plist parse errors preserve original error details
- deleteItemWithoutAuth uses injected FileManager consistently

## Upgrade Instructions

Download the latest release from the GitHub Releases page. Replace the existing LaunchControl.app in your Applications folder.
